ELRC-SHARE RESTful API

ELRC-SHARE provides a RESTful API which returns Language Resource Metadata in JSON format.

API ACCESS

Access is granted by default to repository administrators and to EC members registered and logged in the ELRC-SHARE repository.

The API can be accessed through the SEARCH_API_URL:
https://elrc-share.eu/repository/api/v1/lr/
OR
https://elrc-share.eu/repository/api/v1/lr/search/ to enable advanced search features.

External access, for example through a cURL request, requires an API key, given either in the request url as an authorization (auth) parameter, or in the request header:
As url parameter
SEARCH_API_URL/?auth=API_KEY
As header (cURL)
curl SEARCH_API_URL --header 'Authorization:API_KEY'

The API_KEY is available in the profile page of all EC members (on the main menu: Your profile > Edit your profile).

Haystack Filters

The API leverages the existing ELRC-SHARE repository search functionality using a condensed version of the preconfigured Haystack filters.

All filters are passed as key-value pairs in the q dictionary SEARCH_API_URL/?q=key:value.

Additional filters can be added to the dictionary like: SEARCH_API_URL/?q=key1:value1&q=key2:value2.

Union Queries

The API performs AND search operations by default. For example, the following query
SEARCH_API_URL/?q=language:Norwegian&q=language:Norwegian Nynorsk
will search for resources that contain both Norwegian and Norwegian Nynorsk languages. In order to return all resources that meet either of the specified criteria, we can use the parameter or.
SEARCH_API_URL/?q=lang:no&q=lang:nn&or

Post search filters

Search results can be further narrowed down using custom filters.

  • on: Filters resources created on the specified date formatted as yyyy-mm-dd. e.g. &on=2017-03-01
  • before: Filters resources created before the specified date. e.g. &before=2017-03-01
  • on_before: Filters resources created before the specified date (inclusive). e.g. &on_before=2017-03-01
  • after: Filters resources created after the specified date. e.g. &after=2017-03-01
  • on_after: Filters resources created after the specified date (inclusive). e.g. &on_after=2017-03-01
  • span: Filters resources created within the specified timespan (comma seperated). e.g. &span=2016-03-01,2016-09-01
  • processed: Filters resources that are products of processing. e.g. &processed
  • validated: Filters resources that have been validated (contain a validation report). e.g. &validated
  • cleared: Filters resources for which legal clearance is finalized. e.g. &cleared
  • limit: The API paginates results for faster responses. Each page limits results to 20 by default, unless stated otherwise using the limit filter. e.g. &limit=15
    Setting the value of limit to 0, the API will return all results (if less than 1000) in one page which might take longer to respond.
The results returned by the API can also be sorted by creation date, using the sort parameter with values latest or earliest. e.g &sort=latest
